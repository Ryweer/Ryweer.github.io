##  `rank()` 算子详解

### 一、rank() 的精确含义

#### 1.1 定义
`rank(x)` 是一个**横截面算子（Cross-Sectional Operator）**，用于对股票池内所有股票在单个交易日的数值进行排序和标准化。

#### 1.2 工作流程
1. 锁定时间 → 仅看单个交易日  
   ↓  
2. 全市场排序 → 按 x 的计算值从小到大排序  
   ↓  
3. 均匀映射 → 排名位置转换为 (0, 1) 区间  
   ↓  
4. 每日独立 → 第二天重新排序、重新映射


#### 1.3 直观示例

假设当日股票池有 5 只股票：

| 股票 | `-returns` 原始值 | 排序位置 | `rank()` 输出 |
|------|------------------|----------|---------------|
| A    | -2.0%            | 1        | 0.00          |
| E    | -1.5%            | 2        | 0.25          |
| C    | -0.1%            | 3        | 0.50          |
| B    | +0.5%            | 4        | 0.75          |
| D    | +1.2%            | 5        | 1.00          |

> **注**: BRAIN 内部实际映射公式为 `(排名 - 1) / (有效股票数 - 1)`，缺失数据自动剔除不参与排序。

---

### 二、为什么 Alpha 几乎必须用 rank()

#### 2.1 核心作用对比表

| 作用 | 量化意义 | 不使用 rank() 的后果 |
|------|----------|----------------------|
| **① 消除量纲与极端值** | 将不同价格、不同波动率的股票拉到同一尺度 | 高价股或单日暴涨/暴跌股会主导信号，组合失衡 |
| **② 天然多空平衡** | 输出均值恒为 0.5，做多高分/做空低分自动对冲 | 信号偏向单边，暴露方向性风险，Sharpe 骤降 |
| **③ 适配平台组合优化器** | BRAIN 底层按排名分配权重（如排名 1% 分配 1% 权重） | 优化器无法解析原始值分布，回测结果失真 |
| **④ 保留相对强弱逻辑** | 量化本质是"抓相对优势"，而非预测绝对价格 | 原始值受宏观/行业β干扰大，难以剥离 Alpha |
| **⑤ 降低换手噪音** | 排名变化比原始值跳变更平滑 | 信号频繁翻转，Turnover 飙升，交易成本吞噬收益 |

#### 2.2 关键结论

> ✅ **rank() 不创造预测能力，但它是让原始逻辑变成可交易信号的必要转换器。**
>
> 在 BRAIN 中， **90% 以上可提交 Alpha 的最外层都是 rank()** 。

### 三、BRAIN 中的 Rank 家族

#### 3.1 算子对比表

| 算子 | 作用范围 | 典型用法 | 适用场景 |
|------|----------|----------|----------|
| `rank(x)` | 全市场横截面 | `rank(returns)` | 基础排序、全市场选股 |
| `group_rank(x, sector)` | 分组内横截面 | `group_rank(roe, sector)` | 行业中性化，避免"全买科技股" |
| `ts_rank(x, 20)` | 单股票时间序列 | `ts_rank(close, 20)` | 动量/趋势判断（过去20天相对位置） |
| `zscore(x)` | 横截面标准化 | `zscore(pe)` | 假设数据正态分布时使用（对异常值敏感） |
| `scale(x)` | 线性缩放至和为1 | `scale(volume)` | 权重分配，较少直接用于信号构建 |

#### 3.2 使用口诀
想抓全市场相对强弱 → rank()<br>
想避免行业暴露 → group_rank(..., industry)<br>
想看个股近期位置 → ts_rank(..., d)

### 四、实战避坑指南

#### 4.1 常见错误与正确做法

| 常见错误 | 正确做法 | 原因 |
|----------|----------|------|
| `rank(rank(x))` | 直接用 `rank(x)` | 二次排名结果不变，浪费计算资源 |
| 未处理缺失值直接 rank | BRAIN 自动忽略 NaN，无需手动处理 | 平台已优化，强行填 0 会扭曲排序 |
| 用 rank() 替代中性化 | 外层仍需在 Properties 勾选 `Sector/Industry Neutralization` | rank() 只做排序，不消除系统性风险 |
| 原始值含未来数据仍加 rank | 必须配合 `delay(x, 1)` | rank() 不防未来函数，输入错则输出错 |
| 多因子直接相加不 rank | `rank(w1*factor1 + w2*factor2)` | 各因子量纲不同，必须先对齐再合并 |

#### 4.2 黄金法则

> ⚠️ **rank() 仅处理当前截面数据，不跨时间维度。**
> 
> 如需时间序列排名，请使用 `ts_rank(x, d)`。

### 五、教程案例解析

#### 5.1 当前教程任务

```python
rank(-returns)
```
| 模块 | 作用 | 量化含义 |
|----------|----------|------|
| `returns` | 股票当日收益率 | 原始数据字段 |
| `-` | 取反运算符 | 表达均值回归逻辑：昨日涨得多的今日看跌，昨日跌得多的今日看涨 |
| `rank()` | 横截面排名算子 | 每天将所有股票的 -returns 值从小到大排序，并映射到 [0, 1] 均匀分布 |

#### 5.3 策略表达
`"每天选出昨日跌幅最大（-returns 最大）的股票做多，昨日涨幅最大的股票做空。"`  
这就是经典的**短期均值回归（Mean Reversion）**逻辑。
#### 5.4 加上 rank() 后的优势
✅ 信号被压缩到 [0,1]，平台可安全分配权重<br>
✅ 每日多空数量自动平衡，不赌大盘方向<br>
✅ 为后续加 decay、group_neutralize、多因子融合打下基础

## 📉 `decay_linear` 算子详解
### 🎯 一、核心定义

`decay_linear(x, d)` 是 BRAIN 平台内置的时间序列算子，用于对过去 `d` 天的信号进行**线性衰减加权平均**。

- **核心作用**：平滑信号跳变，降低换手率（Turnover），过滤短期市场噪音。
- **输出特性**：经归一化处理，输出值域与输入 `x` 保持一致，不改变信号量纲。

---

### 🔢 二、数学原理：为什么除以 6？（归一化机制）

当 `d = 3` 时，算子分配原始权重为 `[3, 2, 1]`。为保证加权结果不改变原始信号尺度，必须进行**权重归一化（Normalization）**。

| 时间偏移 | 原始权重 | 归一化权重 | 计算过程 | 影响力占比 |
|----------|----------|------------|----------|------------|
| 今日 (t=0) | 3 | 3/6 | 50% | ✅ 最近数据权重最高 |
| 昨日 (t=1) | 2 | 2/6 | ≈ 33.3% | |
| 前日 (t=2) | 1 | 1/6 | ≈ 16.7% | |
| **合计** | **6** | **1.00** | 6/6 | ✅ 权重和恒为1 |

> 💡 **核心目的**：归一化确保输出与输入量纲一致，避免信号被意外放大/缩小，使平台组合优化器能正确解析权重分配。

---

### 🧮 三、通用计算公式

#### 1. 权重序列
`W = [d, d-1, d-2, ..., 2, 1] （共 d 项，今日权重最大）`

#### 2. 权重总和（等差数列求和）
`Sum = 1 + 2 + ... + d = d × (d + 1) / 2`

#### 3. 归一化权重（第 k 天前，k=0 表示今日）
`weight_k = (d - k) / [d × (d + 1) / 2]`

#### 📊 常见 `d` 值对照表

| d | 权重总和 | 今日权重 | 昨日权重 | 适用场景 |
|---|----------|----------|----------|----------|
| 2 | 3 | 66.7% | 33.3% | 高灵敏度，低平滑 |
| 3 | 6 | 50.0% | 33.3% | ✅ 默认推荐，平衡型 |
| 5 | 15 | 33.3% | 26.7% | 中等平滑，有效降换手 |
| 10 | 55 | 18.2% | 16.4% | 高平滑，趋势跟踪 |

---

### 📐 四、实战计算示例

假设某股票连续 3 天的原始信号值：`[0.2, 0.5, 0.8]`（Day-2 → Day-0）

**计算过程：**

```python
decay_linear(signal, 3) 
= (3×0.8 + 2×0.5 + 1×0.2) / 6
= (2.4 + 1.0 + 0.2) / 6
= 3.6 / 6
= 0.6  # 今日输出的平滑信号
```
效果解读： 今日原始信号 0.8 被历史趋势拉低至 0.6，融合了过去信息。这能有效避免单日噪音导致组合频繁调仓，是控制 Turnover 的关键手段。

### ⚖️ 五、衰减类型对比
| 衰减类型 | 权重分布特征 | 适用场景 | BRAIN 支持情况 |
|---|----------|----------|----------|
| linear | 均匀递减<br>`[d, d-1, ..., 1]` | ✅ 默认推荐，平衡灵敏度与平滑度 | ✅ 内置 `ts_decay_linear` |
| exponential | 指数递减，近期权重极高 | 高频/超短线，极度看重最新信号 | ❌ 需手动实现 |
| flat | 等权重<br>`[1, 1, ..., 1]` | 简单移动平均，滞后性高 | ✅ 等价于 `ts_mean` |
### 🛠️ 六、BRAIN 实战使用指南

#### 6.1 最佳实践
- 标准结构：通常作为 Alpha 表达式的最后一层：`decay_linear(rank(...), d)`

#### 6.2 参数调优指南
- Turnover > 0.6 → 增大 d（如 3 → 5）
- Turnover < 0.3 且收益滞后 → 减小 d（如 3 → 2）
- 平台设置配合：若表达式已含 decay_linear，Properties 面板中的 Decay 建议设为 0，避免双重衰减导致信号过度迟钝。

#### 6.3 常见误区
- ❌ 直接对原始价格使用：`decay_linear(close, 5)` → 量纲无意义，应作用于排名或标准化后的信号
- ❌ d 值过大（如 >20）：信号过度平滑，丧失预测时效性，Sharpe 反而下降
- ❌ 未做 rank() 直接衰减：多因子量纲未对齐时，衰减会放大原始尺度差异
### 📌 七、核心记忆口诀
"线性衰减降换手，权重归一保量纲；
d选三五最均衡，外层包裹是王道。"
## ts_std_dev(x, d) 算子详解
### 📊 什么是 ts_std_dev？
ts_std_dev(x, d) 是一个时间序列算子，用于计算数据系列 x 在过去 d 天的标准差（Standard Deviation）。
### 核心作用
衡量波动性：标准差越大，说明数据波动越剧烈
评估稳定性：标准差越小，说明数据越稳定
风险度量：在量化交易中，常用于衡量股票或策略的风险水平
### 🧮 计算原理（5天示例）
假设有5天的股票日收益率：
```python
x = [0.01, 0.02, -0.01, 0.00, 0.03]
即：[1%, 2%, -1%, 0%, 3%]
```
### 计算步骤：
#### 1️⃣ 计算均值（Mean）
`(0.01 + 0.02 + (-0.01) + 0.00 + 0.03) / 5 = 0.01`
#### 2️⃣ 计算每个值与均值的平方偏差
```python
(0.01 - 0.01)² = 0
(0.02 - 0.01)² = 0.0001
(-0.01 - 0.01)² = 0.0004
(0.00 - 0.01)² = 0.0001
(0.03 - 0.01)² = 0.0004
```
#### 3️⃣ 计算平均平方偏差
`(0 + 0.0001 + 0.0004 + 0.0001 + 0.0004) / 5 = 0.0002`
#### 4️⃣ 取平方根
`√0.0002 ≈ 0.0141`
结果：ts_std_dev(x, 5) ≈ 0.0141（即1.41%）
### 💡 实际应用场景
#### 1. 衡量股票波动性
```python
ts_std_dev(returns, 21)
```
计算21天（约1个月）的滚动标准差，作为股票波动性的代理指标。
#### 2. 波动率调整信号
```python
rank(returns / ts_std_dev(returns, 20))
```
用20天波动率标准化收益率，得到风险调整后的收益信号。
#### 3. 低波动策略
```python
rank(-1 * ts_std_dev(returns, 60))
```
做多低波动股票（标准差小），做空高波动股票。
### 📈 解读标准差数值
| 标准差大小 | 含义 | 投资策略 |
|---|----------|----------|
| 低 | 价格稳定，波动小 | 适合防御型策略 |
| 高 | 价格波动剧烈 | 适合趋势/动量策略 |
### ⚠️ 注意事项
1.窗口选择：
> d=20：约1个月交易日<br>
> d=60：约3个月（季度）<br>
> d=252：约1年（年度波动率）<br>

2.与 rank() 配合使用：
```python
rank(ts_std_dev(returns, 20))  # ✅ 推荐
ts_std_dev(returns, 20)        # ❌ 量纲不统一
```
3.避免除以零：
如果 d 天内数据不足或全为相同值，标准差可能为0，需要处理。
### 🎯 总结
`ts_std_dev` 是量化研究中最基础且最重要的风险度量工具，常用于：<br>
✅ 波动率过滤<br>
✅ 风险调整收益<br>
✅ 低波动因子构建<br>
✅ 止损/风控阈值设定

### 📊 六大实际应用场景

#### 1️⃣ 数据预处理：标准化与去量纲

| 运算 | 应用场景 | 代码示例 | 量化意义 |
|------|----------|----------|----------|
| `/` | 价格归一化 | `price / price.shift(20)` | 消除绝对价格影响，便于跨股票比较 |
| `-` + `/` | Z-Score标准化 | `(x - ts_mean(x,20)) / ts_std(x,20)` | 统一因子分布，提升模型稳定性 |
| `*` | 单位转换 | `volume * price / 1e6` | 将成交量转换为成交额（百万元） |
| `**` 或 `pow()` | 非线性变换 | `(returns + 1)**252 - 1` | 日化收益率→年化收益率 |

> 💡 **关键**：预处理质量直接决定因子有效性，算术运算是数据清洗的第一步。

#### 2️⃣ 因子构建：多因子线性/非线性组合

##### 🔹 线性组合（加权求和）
```python
# 经典三因子模型简化版
alpha = 0.4 * value_factor + 0.3 * momentum_factor + 0.3 * quality_factor
```

##### 🔹 非线性组合（交互效应）
```python
# 价值×动量：低估值且强势的股票
alpha = (pe_rank < 0.3) * (momentum_rank > 0.7)  # 逻辑运算返回0/1，*实现与逻辑

# 波动率调整的收益：高波动惩罚
alpha = returns / (volatility + 1e-6)  # 加小值防除零
```

##### 🔹 条件加权（动态调整）
```python
# 牛市加重动量权重，熊市加重价值权重
market_trend = ts_mean(market_return, 60)
alpha = (market_trend > 0) * (0.7*momentum + 0.3*value) + \
        (market_trend <= 0) * (0.3*momentum + 0.7*value)
```

> 🎯 **本质**：算术运算符实现因子间的**加权、交互、条件选择**，是因子合成的核心工具。

#### 3️⃣ 信号生成：阈值过滤与信号强度

| 运算 | 应用 | 示例 | 效果 |
|------|------|--------|------|
| `>` / `<` + `*` | 信号过滤 | `signal = (rank > 0.8) * raw_score` | 仅保留顶部20%股票，其余置0 |
| `abs()` | 信号强度 | `signal_strength = abs(z_score)` | 用绝对值衡量偏离程度 |
| `round()` / `floor()` | 离散化 | `position_level = floor(signal * 10) / 10` | 将连续信号转为10档仓位 |
| `max()` / `min()` | 截断处理 | `clipped = max(min(x, 3), -3)` | 限制极端值，控制尾部风险 |

```python
# 实战：三重过滤信号
raw_signal = rank(momentum)  # [0,1]区间
filtered = (raw_signal > 0.7) * (volatility < ts_median(volatility, 252))  # 逻辑与
final_signal = filtered * raw_signal * 2 - 1  # 映射到[-1,1]，支持多空
```

#### 4️⃣ 风险控制：仓位计算与风险平价

##### 🔹 基础仓位管理
```python
# 等权重
position = 1 / n_stocks

# 信号强度加权
position = signal / sum(abs(signal))  # 归一化，总敞口=1

# 波动率倒数加权（风险平价思想）
position = (1 / volatility) / sum(1 / volatility)
```

##### 🔹 风险约束
```python
# 单股票仓位上限
position = min(position, 0.05)  # 不超过5%

# 行业中性调整（简化版）
industry_exposure = group_sum(position, industry)
position = position - (industry_exposure - 1/n_industries) / n_stocks_in_industry
```

##### 🔹 杠杆控制
```python
# 总敞口限制
gross_exposure = sum(abs(position))
if gross_exposure > max_leverage:
    position = position * (max_leverage / gross_exposure)  # 等比例缩放
```

> ⚠️ **注意**：风险控制中的算术运算需考虑**交易成本、流动性约束**等现实因素。

| 指标 | 算术表达式 | 量化意义 |

| 累计收益 | `(1 + returns).prod() - 1` | 策略整体盈利能力 |

```python
# BRAIN平台示例：计算滚动夏普比率
rolling_sharp = ts_mean(returns, 60) / ts_std(returns, 60) * (252**0.5)
```

#### 🔢 数值稳定性
```python
# ❌ 危险：除零风险

ratio = a / (b + 1e-10)
```

#### 📉 极端值处理
```python
# Winsorize：截断上下1%分位数
lower = ts_percentile(x, 200, 1)
upper = ts_percentile(x, 200, 99)
x_clipped = max(min(x, upper), lower)
```

#### 🔄 运算顺序与括号
```python
# ❌ 歧义：优先级可能不符合预期
signal = a + b * c > threshold

# ✅ 明确：用括号指定顺序
signal = (a + (b * c)) > threshold
```

#### 🌐 横截面与时间序列混合
```python
# 注意：先时序计算，再横截面排名
# ✅ 正确：每只股票先算20日均值，再当日排名
signal = rank(ts_mean(x, 20))

# ❌ 错误：先排名再时序平均（逻辑不同）
signal = ts_mean(rank(x), 20)
```
