## 1. 数据集总览与架构

### 数据分层架构

BRAIN数据集按信息层级从底向上分为5层：

```
Layer 0  pv1        — 行情数据(OHLCV/市值/ADV)      <- 所有Alpha的原材料
Layer 1  fundamental6  analyst4  fundamental2  — 财务报表 + 分析师预测
Layer 2  model16     — 已排名评分（直接用于排序）
Layer 3  model51     — 系统性风险（BETA/相关性）
Layer 4  news12/18  option8/9  pv13  socialmedia12/8  — 另类数据
Layer 5  univ1       — Universe定义（选股范围边界）
```

**核心原则**：层级越高，数据越加工（已排名/已平滑）；层级越低，数据越原始（需自己做中性化/标准化）。

## 2. 字段价值分析

### 核心发现摘要

基于2,627个字段的统计分析，核心结论如下：

| 结论 | 数据支撑 |
|---|---|
| **pv1是整个平台的原子层** | pv1的23个字段被271.6万个Alpha引用；每个Alpha几乎都会用到close/volume/cap等行情字段 |
| **分析师预期revision是顶级Alpha因子** | analyst4的anl4_adjusted_netincome_ft被3.4万个Alpha引用，排名全球第19 |
| **约20%的字段具有极高覆盖率（90-100%）** | 这部分字段覆盖全市场，可直接用于Universe全局选股 |
| **46%的字段被100~1000个Alpha引用** | 中等热度字段是策略差异化的主要来源 |
| **fundamental6是基本面因子的主战场** | 886个字段，涵盖资产、负债、收入等全科目 |

### 覆盖率分析

覆盖率决定了字段在全市场的可用性，是样本选择偏差的关键来源。

| 覆盖率区间 | 字段数 | 占比 | 说明 |
|---|---|---|---|
| 90-100% | 511 | 19.5% | 覆盖全市场，可直接用于Universe全局选股 |
| 70-90% | 426 | 16.2% | 高覆盖率，样本偏差小，适合大部分Alpha |
| 50-70% | 1263 | 48.1% | 中等覆盖率，需注意行业/市值偏差 |
| 30-50% | 322 | 12.3% | 低覆盖率，仅适合特定行业或大中市值选股 |
| 10-30% | 94 | 3.6% | 窄覆盖，需严格限制选股范围 |
| <10% | 11 | 0.4% | 极窄覆盖，慎用，可能产生极端偏差 |

**实战建议**：在做Alpha公式时，优先选择cov>=70%的字段；若必须使用低覆盖率字段，需在universe()层面限制只在大中市值Universe内计算。

### 字段命名规律

BRAIN字段名遵循统一的前缀/后缀规范：

| 类别 | 命名模式 | 含义 | 示例 |
|---|---|---|---|
| 实际值 | `actual_` | 已披露的实际财务数据 | actual_eps_value_quarterly |
| 预测值 | `_ft` / `forecast_` | 分析师对未来财务的预测 | anl4_af_eps_value |
| 修正信号 | `_revision` | 分析师预测的修正方向 | _revision_rank |
| 均值/中位数 | `_mean` / `_median` | 多个估计值的集中趋势 | anl4_afv4_eps_mean |
| 最高/最低 | `_high` / `_low` | 估计值的乐观/悲观边界 | _high - _low |
| 标准差 | `_std` | 分析师预测的分歧程度 | eps_std |
| 数量 | `_number` | 样本数量 | _number |
| 同比/环比 | `_yoy` / `_mom` | 时间维度的变化率 | sales_yoy |
| 排名 | `_rank` | Universe内百分位排名 | rank(pe_ratio) |
| 日均成交量 | `adv{N}` | N日平均日成交量 | adv20 |
| 历史波动率 | `historical_volatility_{N}` | N日历史波动率 | historical_volatility_10 |
| 隐含波动率 | `implied_volatility_*` | 由期权价格反推的未来波动率 | implied_volatility_call_10 |
| 行业分类 | `sector` / `subindustry` | GICS行业分类 | subindustry |

## 5. 实战策略构建参考

基于字段分析，总结以下经过社区大量验证的Alpha构建范式：

| 策略类型 | 子类 | 参考公式 | 逻辑说明 |
|---|---|---|---|
| 规模因子 | 市值因子 | `rank(-log(cap))` | 做多小市值、做空大市值 |
| 价值因子 | 账面市值比 | `rank(book/cap)` | 做多低估值、做空高估值 |
| 盈利因子 | 盈利质量 | `rank(operating_income/assets)` | 做多高盈利质量、做空低盈利质量 |
| 分析师预期修正 | Revision Momentum | `anl4_adjusted_netincome_ft_revision` | 分析师上调预测 → 正Alpha |
| 分析师分歧度 | 预测离散度 | `anl4_*_eps_std` | 高分歧 → 高风险/机会 |
| 流动性因子 | 成交量异常 | `rank(volume/adv20)` | 成交量异常放大 → 短期动能 |
| 波动率因子 | 低波动异象 | `rank(-historical_volatility_20)` | 低波动股票长期跑赢 |
| 市场Beta因子 | 系统性风险 | `rank(beta_last_30_days_spy)` | 高Beta在牛市更有弹性 |
| 行业轮动 | 子行业动量 | `group_rank(returns, subindustry)` | 行业内部相对强弱轮动 |
| 新闻事件驱动 | Buzz异象 | `scl12_buzz - ts_mean(scl12_buzz, 20)` | 异常关注 → 价格惯性 |
| 分析师覆盖数 | 信息代理 | `rank(anl4_afv4_eps_number)` | 覆盖分析师越多 → 定价越充分 |

## 6. 使用建议总结

| 优先级 | 建议 | 代表字段 |
|---|---|---|
| 必须掌握 | pv1行情字段是所有Alpha的基础 | close, volume, cap, returns, adv20, sector |
| 高频使用 | fundamental6财务报表是价值因子的核心 | assets, liabilities, equity, operating_income |
| 高频使用 | analyst4分析师预期revision是顶级信号 | anl4_adjusted_netincome_ft, revision_rank |
| 重要补充 | pv13关系数据用于供应链策略 | primary_sector_focused_company_count |
| 重要补充 | socialmedia12情绪数据用于另类Alpha | scl12_buzz, sentiment |
| 需注意 | 使用前务必检查覆盖率cov | fundamental2数据覆盖率偏低 |
| 需注意 | 财务数据有1~2个月披露滞后 | analyst4预测数据优于fundamental6 |
